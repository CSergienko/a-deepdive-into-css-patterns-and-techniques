Part One

The DNA of HTML and CSS

One cannot talk about CSS without talking about HTML.

These two technologies are very closely tied to one another, so throughout this presentation we will be discussing aspects of both.

The Rise of the Responsive Web

This USED to be the web. Everything was made for the desktop screen.

When I started, a lot of websites were chopped up images living inside of HTML tables.

Browser support was awful. IE6 and 7 were still around. IE8 was only just starting to die, and IE9 was new and shiny.

Then this happened. The release of the iPhone resulted in a flood of smart phones and tablets on the market.

In emerging markets, these devices were and are still the primary way most of the population accesses the internet.

Quite simply, you could not build for the web anymore without supporting mobile.

Now we're entering this phase. Everything is smart and internet capable.

Cross-browser support hasn't gone away, it's just changing year on year.

Making a usable experience across all of this is hard.

  • Differences in browser engines
  • Differences in OSes
  • Differences in device capabilities

Cross-browser differences used to be a lot worse, but they're still there.

OSes have different ways of managing things like font rendering, color space, etc.

Across devices, issues like dealing with different pixel densities and low specs (particularly for mobile) is common.

Progressive Enhancement

Progressive enhancement is a design philosophy that provides a baseline of essential content and functionality to as many users as possible, while delivering the best possible experience only to users of the most modern browsers that can run all the required code.

Progressive enhancement is a design philosophy that provides a baseline of essential content and functionality to as many users as possible, while delivering the best possible experience only to users of the most modern browsers that can run all the required code.

  • Start with accessible, well structured and semantic HTML content.
  • Do as much as you can with CSS.
  • Use JavaScript to enhance the experience.

The basic idea is that, if you start by building a usable experience for the crappiest platform you have to support, everything else after that is easier.

Regardless of whether you're using server-side templates, doing serverless, using Angular, Vue, React, Svelte, or whatever else, these fundamentals are still at play.

Take advantage of the built-in fault tolerance of HTML and CSS, and use it to your advantage!

HTML using unsupported tags will still work, and they will be treated as divs.

CSS will ignore any property it doesn't support.

No exceptions thrown, they just keep on going.

There is a reason that websites from the 1990s still work in modern browsers.

A core principle of both HTML, CSS and JavaScript is "don't break the internet". The old will always continue working, and using newer features will not break the old ones.

This is also why CSS in particular can be so unintuitive and confusing.

Part Two

A recap of how CSS does things

Types of Selectors

How you decide which elements to target.

Element Selectors

h1 {
  color: red;
}

h1, h2, h3, h4, h5, h6 {
  font-weight: 900;
  margin: 1.5em 0 1em;
}

Pseudo-Elements

blockquote::before {
  content: open-quote;
}

blockquote::after {
  content: close-quote;
}

Class Selectors

.foo {
  color: red;
}

.foo:hover,
button.foo:focus {
  color: green;
}

Pseudo-Classes

button:hover,
button:active,
button:focus {
  color: red;
}

table tr:nth-child(2n) {
  background: #f3f3f3;
}

[type="radio"]:checked {
  transform: scale(5);
}

ID Selectors

#foo {
  color: red;
}

#foo:hover,
button#foo:focus {
  color: green;
}

Attribute Selectors


                
// Just the attribute
[class],
[id],
[data-mycustomattr],
[title] {
  color: red;
}

// Exact value match
[class="bar"],
[id="foo"],
[data-mycustomattr="fizzbuzz"],
[title="I'm a button, yo."] {
  color: red;
}

// Value anywhere in string (even as a substring)
[class*="baz"] {
  color: red;
}

// Exact value, or followed by a -
[class|="baz"] {
  color: red;
}

The Wildcard/Universal Selector

*,
*::before,
*::after {
  box-sizing: border-box;
}

Combinators

// Descendant
h1 small {
  font-size: 25%;
}

// Child
ul > li {
  line-height: 1.2;
}

// Adjacent Sibling
li + li {
  margin-top: 10px;
}

// General Sibling
p ~ span {
  color: gold;
}
